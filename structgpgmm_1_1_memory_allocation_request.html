Detailed Description

◆ Alignment

uint64_t gpgmm::MemoryAllocationRequest::Alignment

Requested alignment, in bytes, of the allocation.

This is the alignment value of the memory block, which is not necessarily a multiple of the allocated size. For example, a 128-byte alignment means the memory block offset must be a multiple of 128. But the block size may only be 64 bytes, leaving the other 64 bytes allocated but unused.

◆ NeverAllocate

bool gpgmm::MemoryAllocationRequest::NeverAllocate

Request to never create underlying memory.

Used to check whether or not the memory allocation could succeed, without requiring actual memory to be created. Or used to ensure the requested allocation memory ONLY comes from the existing "working set" of memory (eg. pool), created by previous allocations.
